## 1. 개요

두 스킬 모두 영상에서 **무음 구간**과 **필러 단어**를 자동으로 제거하는 기능을 제공합니다. 하지만 **무음을 감지하는 방식**과 **처리 아키텍처**가 근본적으로 다릅니다.

| 구분 | AI4PKM Video Cleaning | CatchUpAI Video Edit |
|------|----------------------|---------------------|
| **핵심 방식** | 단어 타임스탬프 기반 | 음량(데시벨) 기반 |
| **API 의존성** | OpenAI Whisper API (유료) | 로컬 처리 (무료) |
| **스크립트 구성** | 2단계 (transcribe → edit) | 1단계 (통합) |
| **편집 철학** | 보수적 (자연스러운 결과) | 공격적 (최대 단축) |

# 단어 사이 간격으로 무음 감지
for i in range(len(words) - 1):
    current_end = words[i]['end']      # 현재 단어 종료 시간
    next_start = words[i + 1]['start'] # 다음 단어 시작 시간

## 3. 무음 감지 방식 비교

### 3.1 AI4PKM: 단어 타임스탬프 기반

**원리**: OpenAI Whisper API가 음성을 인식하여 각 단어의 시작/종료 시간을 반환합니다. 이 메타데이터를 분석하여 **단어와 단어 사이의 간격**이 threshold 이상인 구간을 "무음"으로 판단합니다.

**특징**:
- 말하는 중간의 자연스러운 쉼(호흡, 생각)은 유지됨
- "단어가 없는 구간"만 무음으로 인식
- 음성이 있지만 인식되지 않은 경우 (웅얼거림 등)는 무음으로 처리 안 됨

**예시**:
```
"안녕하세요" (0.0s ~ 1.2s) ... 3초 침묵 ... "반갑습니다" (4.2s ~ 5.5s)
                               ↑
                    이 구간만 무음으로 감지 (단어 사이 간격 3초)
```

### 3.2 CatchUpAI: 음량(데시벨) 기반

**원리**: pydub 라이브러리로 오디오의 음량을 분석하여 **특정 데시벨 이하**인 구간을 "무음"으로 판단합니다.

**특징**:
- 소리가 작은 모든 구간을 감지 (말 사이의 짧은 쉼도 포함)
- 배경 소음이 있으면 무음 감지가 어려울 수 있음
- 말하는 중간에 잠깐 멈추는 부분도 무음으로 감지 가능

**예시**:
```
"안녕" (0.0s ~ 0.5s) [0.3초 쉼] "하세요" (0.8s ~ 1.2s) ... 3초 침묵 ...
                      ↑                                      ↑
              이 구간도 감지 가능 (음량 -40dB 이하)    이 구간도 감지
```

### 3.3 감지 방식 차이의 결과

| 상황 | AI4PKM | CatchUpAI |
|------|--------|-----------|
| 말 사이의 긴 침묵 (3초) | 감지 O | 감지 O |
| 말 중간의 짧은 쉼 (0.5초) | 감지 X | 감지 O (설정 따라) |
| 낮은 목소리로 말하는 중 | 감지 X | 감지할 수 있음 |
| "음..." 하며 생각하는 중 | 감지 X (단어로 인식) | 감지할 수 있음 |
| 배경 음악이 있는 침묵 | 감지 O | 감지 X |

## 5. FFmpeg 편집 방식 비교

### 5.1 AI4PKM: 세그먼트 분할 후 병합

```bash
# 각 세그먼트를 개별 파일로 추출
ffmpeg -ss 0.0 -i input.mkv -t 5.0 segment_0001.mp4
ffmpeg -ss 8.0 -i input.mkv -t 3.0 segment_0002.mp4
...

# concat demuxer로 병합
ffmpeg -f concat -safe 0 -i concat_list.txt -c copy output.mkv
```

**특징**:
- 각 세그먼트마다 FFmpeg 호출
- 세그먼트 수가 많으면 오버헤드 증가
- drawtext 필터로 "Skipping X secs" 텍스트 오버레이 가능

### 5.2 CatchUpAI: select 필터 사용

```bash
# select 필터로 유지할 구간만 선택
ffmpeg -i input.mkv \
  -filter_complex "[0:v]select='between(t,0,5)+between(t,8,11)+...',setpts=N/FRAME_RATE/TB[v]; \
                   [0:a]aselect='between(t,0,5)+between(t,8,11)+...',asetpts=N/SR/TB[a]" \
  -map "[v]" -map "[a]" output.mkv
```

**특징**:
- 100개 구간씩 묶음 처리 (명령어 길이 제한 대응)
- 한 번의 FFmpeg 호출로 여러 구간 처리
- 긴 영상에서 효율적 (이전 concat 필터 대비 개선)

## 9. 파라미터 가이드

### 9.1 AI4PKM 파라미터

```bash
python edit_video_remove_pauses.py video.mp4 \
  --pause-threshold 1.0 \  # 단어 사이 간격 (초)
  --padding 0.1 \          # 컷 주변 패딩 (초)
  --no-fillers             # 필러 제거 비활성화
```

| 파라미터 | 기본값 | 설명 |
|---------|-------|------|
| `--pause-threshold` | 1.0 | 무음으로 판단할 최소 단어 간격 (초) |
| `--padding` | 0.1 | 컷 앞뒤에 추가할 여유 시간 |
| `--no-fillers` | false | 필러 단어 제거 비활성화 |
| `--preview` | false | 편집 없이 결과 미리보기 |

### 9.2 CatchUpAI 파라미터

```bash
python video-editor.py video.mp4 \
  --remove-silence \
  --threshold -40 \        # 무음 판단 데시벨
  --min-silence 1000       # 최소 무음 길이 (ms)
```

| 파라미터 | 기본값 | 설명 |
|---------|-------|------|
| `--threshold` | -40 | 무음 판단 임계값 (dB) |
| `--min-silence` | 500 | 무음으로 판단할 최소 길이 (ms) |
| `--remove-fillers` | false | 필러 단어 제거 활성화 |
| `--model` | base | Whisper 모델 (필러 감지용) |

### 9.3 동일한 결과를 얻으려면?

두 스킬에서 유사한 결과를 얻기 위한 파라미터 매핑:

- CatchUpAI: 음량 1초 이하 → 761개 감지
