# Clearly 빠른 시작 가이드

## 개요

이 가이드는 Clearly를 처음 사용하는 사람이 BRD를 생성하기까지의 과정을 안내합니다.

**소요 시간**: 10-15분
**필요한 것**: 웹 브라우저, Google 계정

---

## Step 1: 접속 및 로그인

1. https://www.clearlyreqs.com/ 접속
2. **"Get Started"** 또는 **"Login"** 클릭
3. Google 계정으로 로그인

---

## Step 2: 새 프로젝트 생성

대시보드에서 **"Create Project"** 버튼 클릭

### 2-1. Project Title 입력

```
예: Catch Up AI 2026 Homepage
```

- 프로젝트를 식별할 수 있는 간단한 이름
- 나중에 대시보드에서 찾기 쉽게

### 2-2. Initial Idea 입력

```
예: To introduce Catch Up AI's 2026 methodologies and projects
(Vibe Coding, Vibe Learning, Vibe Guiding, AI4PKM) to visitors,
helping them understand and apply AI-powered approaches for
personal knowledge management and software development.
```

- 프로젝트의 목적, 목표, 달성하고자 하는 것 설명
- **최소 20자** 이상 입력
- 가능한 구체적으로 작성

**"Next"** 클릭

---

## Step 3: 모드 선택

### Plain Mode
| 항목 | 내용 |
|------|------|
| **대상** | 비기술자, 비즈니스 이해관계자 |
| **적합한 도구** | V0, Lovable, Bubble, Replit, Bolt |
| **출력물** | BRD + PRD + **Vibe Coding Prompts** |

### Technical Mode
| 항목 | 내용 |
|------|------|
| **대상** | 개발자, 기술 팀 |
| **적합한 도구** | CLI, Cursor, Claude Code, VS Code, IDEs |
| **출력물** | BRD + PRD + **Detailed Task Lists** |

**선택 기준**:
- 노코드/로우코드 도구 사용 → **Plain Mode**
- 전통적 IDE/코딩 도구 사용 → **Technical Mode**
- 잘 모르겠다면 → **Plain Mode** (더 접근하기 쉬움)

**"Create & Start BRD"** 클릭

---

## Step 4: BRD Wizard 진행

AI Assistant가 대화형으로 질문을 합니다.

### 질문 흐름 (예시)

| 순서 | 질문 주제 | 설명 |
|------|----------|------|
| Q1 | **Purpose** | 프로젝트의 목적, 해결하는 문제 |
| Q2 | **Target Audience** | 누구를 위한 제품인지 |
| Q3 | **Key Features** | 핵심 기능 |
| Q4 | **Constraints** | 예산, 기한, 제약사항 |
| Q5+ | **추가 질문** | 프로젝트에 따라 다름 |

### 팁

- **진행률 확인**: 상단에 "Questions answered: X/5+" 표시
- **샘플 답변 활용**: 각 질문마다 3개의 예시 제공
- **완벽하지 않아도 OK**: 나중에 수정 가능

---

## Step 5: BRD 생성 및 검토

모든 질문에 답변하면 BRD가 생성됩니다.

### BRD 구성 요소

- Executive Summary
- Business Objectives
- Target Audience
- Key Requirements
- Success Metrics
- Constraints & Assumptions

### 검토 및 수정

- 생성된 BRD 내용 검토
- 필요시 수정 (Edit 기능)
- 만족스러우면 다음 단계로

---

## Step 6: PRD 생성 (선택)

BRD 완료 후 PRD 생성 가능

### PRD 추가 구성 요소

- Detailed Feature Specs
- User Stories
- Technical Requirements
- Implementation Details

---

## Step 7: 내보내기

### 내보내기 옵션

| 형식 | 용도 |
|------|------|
| **Markdown** | 문서 저장, GitHub |
| **PDF** | 공유, 보관 |
| **Vibe Coding Prompt** | AI 코딩 도구에 바로 사용 |

---

## 자주 묻는 질문

### Q: 답변을 잘못 입력했어요
**A**: BRD 생성 후 Edit 기능으로 수정 가능합니다.

### Q: Plain Mode와 Technical Mode 중 뭘 선택해야 하나요?
**A**: 사용할 개발 도구에 따라 선택하세요. 노코드 도구면 Plain, IDE면 Technical.

### Q: BRD만 만들고 PRD는 나중에 만들어도 되나요?
**A**: 네, BRD 완료 후 언제든 PRD를 추가로 생성할 수 있습니다.

### Q: 팀원과 공유하려면?
**A**: Real-time Collaboration 기능으로 Viewer 또는 Editor 권한으로 초대 가능합니다.
