# Clearly란?

## 한 줄 정의

> **Clearly**는 AI 기반 Q&A 위자드를 통해 BRD(Business Requirements Document)와 PRD(Product Requirements Document)를 생성하는 플랫폼입니다.

---

## 핵심 가치

**"Clear requirements = better AI-generated code"**

AI 코딩 시대에 요구사항의 품질이 결과물의 품질을 결정합니다. Clearly는 누구나 쉽게 전문적인 요구사항 문서를 작성할 수 있도록 도와줍니다.

---

## 작동 방식

### 프로젝트 생성 흐름

```
Step 1: Create New Project
  ↓ Project Title + Initial Idea 입력 (최소 20자)
Step 2: Choose Your Mode
  ↓ Plain Mode 또는 Technical Mode 선택
Step 3: AI Wizard (BRD)
  ↓ 비즈니스 요구사항 Q&A (5+ 질문)
Step 4: Generate PRD
  ↓ 제품 요구사항 상세화
Step 5: Export
  → 문서 내보내기 또는 AI 코딩 도구용 프롬프트 생성
```

---

## 두 가지 모드

### Plain Mode
- **대상**: 비기술자, 비즈니스 이해관계자
- **적합한 도구**: V0, Lovable, Bubble, Replit, Bolt
- **출력물**: BRD + PRD with **vibe coding prompts**
- **특징**: 비즈니스 중심 질문, 접근하기 쉬운 언어

### Technical Mode
- **대상**: 개발자, 기술 팀
- **적합한 도구**: CLI, Cursor, Claude Code, VS Code, IDEs
- **출력물**: BRD + PRD + **detailed technical task lists**
- **특징**: 스프린트 작업 목록, 의존성, 우선순위, 공수 추정

---

## AI Wizard 체험 (BRD)

실제 탐색에서 확인한 BRD Wizard 질문 흐름:

| 순서 | 질문 주제 | 예시 |
|------|----------|------|
| Q1 | **Purpose** (목적) | 이 프로젝트가 해결하는 문제, 달성할 목표 |
| Q2 | **Target Audience** (타겟) | 누구를 위한 제품인지 |
| Q3+ | (추가 질문들) | 기능, 제약사항 등 |

**특징**:
- 진행률 표시 (예: 1/5+, 20%)
- 각 질문마다 3개의 샘플 답변 제공
- 대화형 채팅 인터페이스

---

## 핵심 기능

### 1. AI Wizard
- 대화형 인터페이스로 요구사항 수집
- 단계별 질문을 통해 체계적으로 정보 정리
- 5-10분 내 BRD 초안 생성

### 2. Contextual Tips
- 각 질문에 대한 설명과 샘플 답변 제공
- 비기술자도 이해할 수 있는 가이드
- 장단점 설명으로 의사결정 지원

### 3. Real-time Collaboration
- 팀원 초대 (Viewer / Editor 권한)
- 이해관계자 간 실시간 협업
- 단일 진실의 원천(Single Source of Truth)

---

## Vibe Coding과의 관계

Clearly는 **Vibe Coding** 워크플로우의 핵심 도구입니다:

1. **비전 → 문서화**: Clearly로 아이디어를 BRD/PRD로 변환
2. **문서 → 프롬프트**: 생성된 문서를 AI 코딩 도구용 프롬프트로 활용
3. **프롬프트 → 코드**: AI가 명확한 요구사항을 바탕으로 코드 생성
