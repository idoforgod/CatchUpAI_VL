# 효과적인 Remotion 프롬프트 작성법

## 기본 원칙

좋은 프롬프트는 **구체적**이고 **시각적**입니다. AI가 정확히 무엇을 만들어야 하는지 알 수 있도록 다음 정보를 포함하세요.

## 첫 생성 프롬프트 구조

```
[영상 종류] + [길이] + [핵심 요소] + [스타일/분위기]
```

### 예시

| 수준 | 프롬프트 | 결과 |
|------|---------|------|
| 나쁨 | "영상 만들어줘" | 무엇을 만들지 모름 |
| 보통 | "인트로 영상 만들어줘" | 기본적인 인트로 |
| 좋음 | "5초짜리 채널 인트로. 채널명 spring 등장, 구독 버튼, 어두운 배경" | 구체적 결과 |
| 최고 | "5초짜리 채널 인트로. 어두운 그라데이션 배경에 'Catch Up AI' 글자별 스태거 등장(bouncy spring), 금색 장식 라인, 태그라인 슬라이드인, 네온 구독 버튼 펄스, 페이드아웃 퇴장" | 의도에 가장 가까운 결과 |

## 반복 개선 피드백 패턴

### 패턴 1: 구체적 수정 요청
```
"배경에 회전하는 그라데이션을 추가하고,
등장 애니메이션을 더 화려하게 해줘"
```

### 패턴 2: 레이어 추가 요청
```
"별 반짝임 40개, 빛줄기 4개,
장식 원 3개를 배경에 추가해줘"
```

### 패턴 3: 기법 지정 요청
```
"글자를 하나씩 스태거로 등장시켜줘.
구독 버튼에 네온 글로우 펄스를 추가해줘"
```

## 프롬프트 체크리스트

- [ ] 영상 길이를 명시했는가? (초 단위)
- [ ] 핵심 요소를 나열했는가? (텍스트, 아이콘, 배경 등)
- [ ] 색상/분위기를 지정했는가? (어두운, 밝은, 전문적인 등)
- [ ] 데이터 구조를 언급했는가? (Props로 변경 가능하게)
- [ ] 등장/퇴장 방식을 지정했는가? (spring, 페이드, 슬라이드 등)

## 반복 개선 워크플로우

```
1회차: 기본 구조 생성
  ↓ "배경/색상/레이아웃 변경"
2회차: 비주얼 개선
  ↓ "애니메이션 추가/수정"
3회차: 모션 완성
  ↓ (필요 시) 수동 미세조정
최종: 완성
```

### 실제 적용 사례 (ChannelIntro)

| 반복 | 피드백 | 결과 |
|------|--------|------|
| 1회 | "채널 인트로를 만들어줘" | 기본: spring 등장 + 장식 라인 + 구독 버튼 |
| 2회 | "배경에 그라데이션 추가, 더 화려하게" | 회전 그라데이션, 장식 원, 파티클, 글로우 추가 |
| 3회 | "더 화려하게" | 40개 별, 빛줄기, 글자별 스태거, 네온 글로우, SVG 웨이브 |

## 핵심 팁

1. **한 번에 모든 것을 요구하지 마세요** - 기본 → 피드백 → 개선이 더 효과적
2. **시각적 언어를 사용하세요** - "글로우", "바운스", "슬라이드인" 등 구체적 모션 용어
3. **Props 재사용을 요청하세요** - "데이터를 Props로 받아서 교체 가능하게"
4. **M2/M3에서 배운 용어를 활용하세요** - spring, interpolate, Series, Sequence 등
