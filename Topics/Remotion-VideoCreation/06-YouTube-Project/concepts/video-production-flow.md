# 영상 제작 플로우: 기획 → 제작 → 렌더링

## 전체 플로우

```
1. 기획 (주제, 스토리보드)
   ↓
2. 재활용 가능한 기존 컴포넌트 파악
   ↓
3. 새 컴포넌트 제작 (Core + Skills)
   ↓
4. 통합 컴포넌트 작성 (Series로 순차 배치)
   ↓
5. Studio에서 확인 & 타이밍 조정
   ↓
6. YouTube 최적 설정으로 렌더링
```

## 핵심 패턴: Core + Skills 하이브리드

| 접근 | 사용 시점 | 장점 |
|------|---------|------|
| **Skills (AI 생성)** | 빠른 프로토타이핑, 반복 개선 | 빠름, 다양한 시도 가능 |
| **Core (수동 코딩)** | 구조 설계, 통합, 미세조정 | 정확한 제어, 재사용성 |
| **재활용** | 기존 컴포넌트 + Props 변경 | 가장 효율적 |

## 실제 적용 (VibeLearn AI 소개 영상)

- **재활용 3개**: ChannelIntro(M4), TitleCard(M3), ExplanationScene(M4) → Props만 변경
- **신규 2개**: WorkflowScene, OutroScene → Core 수동 코딩
- **통합**: VibeLearnIntro.tsx → Series로 5개 장면 순차 배치

## 컴포넌트 재활용 전략

1. 기존 컴포넌트가 Zod 스키마 + Props로 설계되어 있으면 → **데이터만 교체**하여 재활용
2. 새 장면이 기존 패턴과 유사하면 → 기존 코드 **복사 후 수정**
3. 완전히 새로운 장면이면 → Core로 직접 코딩 또는 Skills로 생성

## 타이밍 조정 팁

- 텍스트가 많은 장면: 최소 4-5초 (읽을 시간 확보)
- 포인트 설명: 포인트당 4초 권장
- 인트로/아웃트로: 5-6초
- 전체 재생 후 "읽기 어려운 부분"을 찾아 늘리기
